Q2: "How do you troubleshoot a failing integration?"

Framework Response (Demonstrate Systematic Approach):
"I use a structured diagnostic framework:

1. Error Log Analysis (First 10 minutes):
   - Review integration platform logs (Workato/middleware error logs)
   - Identify failure pattern: Is it systematic or intermittent?
   - Check error codes and timestamps to isolate affected records

2. API Endpoint Testing (Next 15 minutes):
   - Use Postman or similar tool to test API endpoints directly
   - Verify authentication tokens haven't expired
   - Check if source system API is responding (sometimes the issue is upstream)

3. Data Mapping Validation (20 minutes):
   - Examine failed records for data format issues
   - Check for field length violations, special characters, null value handling
   - Compare successful vs. failed records to identify patterns

4. Middleware Configuration Review:
   - Verify connection settings haven't changed
   - Check if recent system upgrades affected integration configuration
   - Review transformation logic for edge cases

5. Stakeholder Communication:
   - Notify affected users with ETA for resolution
   - Document root cause and corrective action
   - Implement monitoring alerts to prevent recurrence

Example: At DHGate, we had an intermittent failure in employee sync. Error logs showed 'field length exceeded' errors. I discovered Chinese employee names with 4+ characters were being truncated in iSolved's 'First Name' field (20 char limit). I modified the transformation logic to concatenate First + Middle name fields when needed, resolving the issue permanently."
